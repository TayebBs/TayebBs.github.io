Project Descriptions:

Ragdoll Multiplayer with mirror:

Description:

Ragdoll Multiplayer with Mirror is a unique game that combines ragdoll physics and multiplayer networking to create an exciting and entertaining experience. The game’s main objective is to capture the most flags and fight whoever stands in your way. The ragdoll characters move and behave realistically, adding a level of fun and unpredictability to the gameplay.

As the developer, I was fascinated by the potential of ragdoll physics and wanted to explore how they would behave in a game setting. To further enhance the experience, I also added a multiplayer layer using the Mirror networking framework in Unity.

The result is a game that not only showcases the power of ragdoll physics but also allows players to compete against each other in real-time, adding a level of excitement and unpredictability to the gameplay.

3d Endless runner (Realtime leader board with firebase google)

Description:

This is a 3D Endless Runner game created with Unity that features a rabbit character running through a randomly generated obstacle course.

The game uses the gyroscope for left and right movement, providing a unique and immersive control system. Firebase integration tracks and displays players’ scores in real-time using Firebase’s real-time database, making the game a great example of integrating Firebase with Unity.

Overall, the game showcases the use of Unity, gyroscope controls, and Firebase to create an engaging and challenging gameplay experience for players.

RTS(Real Time strategy) 2D ISOMETRIC MOBILE GAME(PLAY VS AI)

Description:

This is a RTS(Real Time strategy) 2D ISOMETRIC MOBILE GAME created with Unity that features everything that I studied during my 3 years, I tried to showcase as mechanics in the game as i could.

The game begins by prompting the player to choose their culture from three options: Greek, Salyens, or Roman. Upon selection, the player will find themselves on a barren field next to Glanum with six workers and a castle. An AI-controlled enemy culture will be randomly selected from the remaining two options.

To progress in the game, the player must utilize their workers to gather essential resources such as wood, gold, steel, and rocks, and construct their barracks to create and develop their army. The player’s army will consist of three types of soldiers: archers, cavalry, and infantry (foot soldiers).

As the game unfolds, the player must explore the map to locate and engage in combat with their enemy, while also defending their own structures from enemy attacks. Once the player achieves victory against the opposing culture, they will emerge as the ruler of Glanum and emerge triumphant.

This real-time strategy game presents a challenging and immersive experience, requiring the player to think critically and strategically to overcome obstacles and claim victory. With engaging gameplay mechanics, well-crafted visuals, and a richly detailed world, players are guaranteed to enjoy the experience from start to finish.

Mechanics in this game:

  • Fog of war
  • Movement was a bit tricky because we had to make the units not collapse
  • Unit formations(square/circle/triangle)
  • Building mechanic
  • Spawn mechanic
  • Farming or mining
  • combat(melee/ranged)
  • economy system
  • saving system (we had a lot of data to save)

and more…

For the ai in the game:

It was decided to use “the Goal Oriented Action Planning” which was specifically designed for real-time control of autonomous character behavior in games.

The functioning of the AI works following this logic . It has a list of actions that will be performed if it meets a condition using a priority system that determines which action will be performed first  but since there are different types and multiple AI agents  in the game,the AI brain will be able to  perform multiple actions simultaneously.

This type of AI is used in a lot of games, for example F.E.A.R.

Moreover, some actions need to use data from the world state to determine if they are able to run. These preconditions are called procedural preconditions.
